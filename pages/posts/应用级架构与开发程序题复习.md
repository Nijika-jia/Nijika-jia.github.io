这是一个非常典型的 **SSM（Spring + SpringMVC + MyBatis** 整合实验。针对这种类型的期末抽考，老师通常喜欢考察“核心配置”和“关键业务逻辑”。

#### 题目 1：Spring 与 MyBatis 核心整合配置

**题目描述**：请补全 `applicationContext.xml` 中配置 MyBatis `SqlSessionFactory` 和 扫描 Mapper 接口的关键代码。

```xml
<bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource" ... />

<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="________(1)________" ref="dataSource" />
    <property name="configLocation" value="classpath:mybatis-config.xml"/>
    <property name="________(2)________" value="classpath:mapper/*.xml" />
</bean>

<bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
    <property name="________(3)________" value="com.example.dao" />
</bean>

<bean id="transactionManager" class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
    <property name="dataSource" ref="dataSource" />
</bean>

<tx:________(4)________ transaction-manager="transactionManager" />

```

**【参考答案与解析】**

1. **dataSource** (必须把连接池注入给工厂)
2. **mapperLocations** (告诉Spring去哪里找写SQL的xml文件)
3. **basePackage** (告诉Spring去哪个包下找Mapper接口)
4. **annotation-driven** (开启 `<tx:annotation-driven/>` 才能使用 `@Transactional`)

### 第二部分：代码阅读题 (考察逻辑与事务)

#### 题目 1：事务回滚机制分析

**代码片段**：

```java
@Service
public class UserServiceImpl implements UserService {
    @Autowired
    private UserMapper userMapper;

    @Transactional
    public void registerUser(User user) {
        // 1. 插入用户数据
        userMapper.insertUser(user);
        
        // 2. 模拟一个运行时异常
        if (user.getUsername().equals("error_test")) {
            throw new RuntimeException("强制报错");
        }
    }
}

```

**问题**：

1. 如果传入的 `username` 是 "zhangsan"，数据能保存进数据库吗？
2. 如果传入的 `username` 是 "error_test"，第1步插入的数据会存在于数据库中吗？为什么？

**【参考答案】**

1. **能**。正常执行，事务提交。
2. **不能** (或者说会回滚)。
* **原因**：方法加了 `@Transactional` 注解，默认配置下，当抛出 `RuntimeException` (运行时异常) 时，Spring 会触发事务回滚 (Rollback)，之前执行的 `insert` 操作会被撤销，保证数据一致性。



#### 题目 2：MyBatis 动态 SQL 阅读

**代码片段**：

```xml
<select id="findUser" resultType="com.example.pojo.User">
    SELECT * FROM t_user
    <where>
        <if test="username != null and username != ''">
            AND username = #{username}
        </if>
        <if test="phone != null">
            AND phone = #{phone}
        </if>
    </where>
</select>

```

**问题**：
如果调用方法时传入参数 `username = "admin"`, `phone = null`，最终生成的 SQL 语句是什么？

**【参考答案】**
`SELECT * FROM t_user WHERE username = ?` (MyBatis 会自动处理第一个 `AND`，将其去除)。

### 第三部分：程序设计题 (最可能考的核心业务)

老师通常会要求你手写**Service层实现类**或者**Mapper XML**。

#### 题目 1：设计用户注册业务逻辑 (Service层)

**要求**：编写 `UserServiceImpl` 的 `register` 方法。

1. 判断用户名是否已存在（调用 Mapper 查询）。
2. 如果存在，抛出异常或返回错误提示。
3. 对密码进行 MD5 加密（假设有一个 `MD5Utils.encrypt()` 工具方法）。
4. 保存用户。

**【参考代码】**

```java
@Service
public class UserServiceImpl implements UserService {

    @Autowired
    private UserMapper userMapper;

    @Override
    @Transactional(rollbackFor = Exception.class) // 建议加上，以防万一
    public void register(User user) {
        // 1. 参数验证（用户名是否存在）
        User existUser = userMapper.selectByUsername(user.getUsername());
        if (existUser != null) {
            throw new RuntimeException("用户名已存在！");
        }

        // 2. 密码加密 (实验常考点：不能存明文)
        String rawPassword = user.getPassword();
        String encryptedPassword = MD5Utils.encrypt(rawPassword); // 假设的工具类
        user.setPassword(encryptedPassword);

        // 3. 数据持久化
        int rows = userMapper.insert(user);
        if (rows <= 0) {
            throw new RuntimeException("注册失败，请重试");
        }
    }
}

```

### 💡 实验总结与常见问题 (写报告/回答老师提问用)

如果在考试中涉及到“遇到的问题及解决方案”，你可以准备以下几点（选1-2点背下来）：

1. **问题**：MyBatis 报错 `Invalid bound statement (not found)`。
* **原因**：Mapper 接口和 XML 文件没有对应上，或者 XML 文件没有被 Maven 导出（在 `pom.xml` 的 `build` 节点下漏配了 `resources`）。
* **解决**：检查 namespace 是否正确，或者在 pom 中添加 resource 过滤配置。


2. **问题**：事务不生效，异常发生后数据依然插入了。
* **原因**：可能是在 `try-catch` 块中捕获了异常但没有抛出，导致 Spring 认为方法正常结束；或者是数据库引擎（如 MySQL MyISAM）不支持事务。
* **解决**：在 `catch` 块中手动抛出 `RuntimeException`，或者确保数据库表使用的是 `InnoDB` 引擎。


3. **问题**：注入失败 `No qualifying bean of type...`。
* **原因**：忘记在类上加 `@Service` 或 `@Controller` 注解，或者扫描包路径 `base-package` 配置错误。



---

**建议复习策略：**

1. **背诵配置**：`SqlSessionFactoryBean` 和 `MapperScannerConfigurer` 的配置代码。
2. **理解流程**：Controller -> Service -> Mapper -> DB 的数据流向。
3. **关注注解**：`@Autowired`, `@Service`, `@Transactional`, `@Mapper` (或 XML 扫描)。

### 第二部分：程序阅读题 (重点考察逻辑与事务)

#### 题目 1：事务回滚机制判断

**题目描述**：阅读以下代码，回答问题。

```java
@Service
public class UserServiceImpl implements UserService {
    @Autowired
    private UserMapper userMapper;

    @Transactional
    public void registerUser(User user) {
        // 步骤 A: 插入用户数据
        userMapper.insertUser(user);
        
        // 步骤 B: 模拟异常
        if ("admin".equals(user.getUsername())) {
             // 这是一个运行时异常
            throw new RuntimeException("禁止注册管理员账号");
        }
    }
}

```

**问题**：

1. 当传入用户名为 "zhangsan" 时，数据库是否会新增一条记录？
2. 当传入用户名为 "admin" 时，步骤 A 插入的数据最终会保存在数据库中吗？为什么？

**【参考答案】**

1. **会**。没有异常，事务正常提交。
2. **不会**。
* **原因**：方法上加了 `@Transactional` 注解。当代码抛出 `RuntimeException`（运行时异常）时，Spring 事务管理器会捕获该异常并执行 **Rollback（回滚）** 操作，撤销步骤 A 的数据库操作，保证原子性。



---

#### 题目 2：MyBatis 动态 SQL 阅读

**题目描述**：阅读以下 Mapper XML。

```xml
<select id="findUser" resultType="User">
    SELECT * FROM t_user
    <where>
        <if test="username != null and username != ''">
            AND username = #{username}
        </if>
        <if test="status != null">
            AND status = #{status}
        </if>
    </where>
</select>

```

**问题**：如果在 Java 代码中调用 `findUser` 方法，传入参数 `username` 为 null，`status` 为 1。最终执行的 SQL 语句是什么？

**【参考答案】**
`SELECT * FROM t_user WHERE status = ?`
*(解析：MyBatis 的 `<where>` 标签会自动去掉开头的 AND，因为 username 为空，第一个 if 不执行，第二个 if 执行，`<where>` 会把 `AND status...` 处理成 `WHERE status...`)*

### 第三部分：程序设计题 (重点考察业务实现)

根据实验内容第 3、4 点，这是最可能考手写代码的地方。

#### 题目 1：设计“用户登录”的 Service 层逻辑

**题目要求**：
编写 `UserServiceImpl` 中的 `login` 方法。

1. 根据用户名查询用户。
2. 判断用户是否存在，不存在抛出异常。
3. 判断密码是否正确（假设数据库存的是 MD5 加密后的密码，需要对输入密码也进行加密比对）。
4. 验证通过返回 User 对象。

**【参考代码】**

```java
@Service
public class UserServiceImpl implements UserService {

    @Autowired
    private UserMapper userMapper;

    @Override
    public User login(String username, String password) {
        // 1. 根据用户名查询用户 (调用 Mapper)
        User user = userMapper.findByUsername(username);

        // 2. 判断用户是否存在
        if (user == null) {
            throw new RuntimeException("用户不存在");
        }

        // 3. 密码比对 (实验要求：密码加密)
        // 假设有一个工具类 MD5Utils
        String encryptedInputPwd = MD5Utils.md5(password);
        
        if (!user.getPassword().equals(encryptedInputPwd)) {
            throw new RuntimeException("密码错误");
        }

        // 4. 登录成功，返回用户
        return user;
    }
}

```

### 💡 备考小贴士（针对实验总结）

如果考试要求写“遇到的问题及解决方案”（基本要求第4点），你可以背这一个通用的：

* **问题**：启动时报错 `Invalid bound statement (not found)`。
* **原因**：Mapper 接口的方法名与 XML 文件中的 `id` 不一致，或者是 XML 文件没有被 Maven 正确导出（在 target 目录下找不到 XML）。
* **解决方案**：检查接口与 XML 的对应关系；或者在 `pom.xml` 的 `<build>` 标签下配置 `<resources>` 以允许导出 xml 文件。
